# 学び

## テスト駆動

### ステップ

1. やりたいことをバックログに積む
2. 今回実現すべきものを選ぶ
3. やるべきタスクを洗い出す
4. 仕様を文章で書く
5. Red：文章で書かれた仕様をテストコードにする
  ※ 構文エラーは無く、実装すればテストコードが通る状態にする
6. Green：テストコードが通るように実装する
7. Refactor：テストコードが通る状態を保ちつつ、コードを綺麗にする


### 最初のテストコード

- 目的：文章で書かれた仕様メモを実行可能な仕様（テストコード）にする。
- Domainを固めるのを最優先にする。
- ドメインルール（状態遷移や制約などの契約）の検証を優先する（コンストラクタなど生成についてはやりすぎない）
- 仕様カバレッジを優先

### テストの書き方

- Given-When-Thenのスタイルで書く
  - Given：前提条件
  - When：操作
  - Then：期待される結果
- 1テスト1アサーションを心がける。テストコード内で分岐は避ける。
