# TDDとDDDで作る超絶シンプルなタスクアプリ

敢えてTDD・ドメイン駆動で作る。DDDの設計を大切にしつつ、設計が複雑にし過ぎないようTDDで守る。
自分が成功体験を積むためのプロジェクト。あくまで自分のためのプロジェクト。
画面やDB等複雑なところは後回しにする。少しずつ改良していくかもしれないが、ver1ではCLIで完結させる。

要件は .docs/plan/given-ver1.md にある

## 進め方

1. 現在の理解でのTDD・DDDで実装を進め、完成させる。作り切ることが目的。
2. 完成品をAIに評価してもらう。振り返る。
3. 1で作ったものをリファクタリングする。
4. 品質担保用の単体テスト・結合テスト・総合テストを可能な限り追加する。
