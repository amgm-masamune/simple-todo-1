# スプリントについて

## スプリントの流れ

### 1. スプリント開始

1. Backlog から実装するタスクを選び、スプリントバックログに移す
2. requirement.md を作成
3. Git ブランチを作成する（例：`feature/<task-name>`）
4. タスクリストを作成する（スプリントバックログのタスクを詳細に分解）

### 2. スプリント実施

1. TDDテスト
2. 実装
3. (網羅的なテスト作成？)
4. コードレビュー（AIレビューを含む）
5. リファクタリング
6. プルリクエスト作成
7. マージ

### 3. スプリント終了

1. 振り返り（README作成）
2. バックログ整理
